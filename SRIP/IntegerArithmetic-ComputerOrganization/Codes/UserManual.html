Basic theory behind the application

1 .This application is basically designed to make the user understand the difference and similarities between different arithmetic representation of integers.

2 .The key idea is to be able to observe and understand that some of the arithmetic representation behave isomorphically
hence any of them could be interchangebly used at the backend without the knowledge of the user.

3 .JavaScript has been used to provide an interctive experience for the user


How to use the application

Binary Addition-  Select any two rows that you want to add and press the ADD button. Note that the contents of the table have been generated dynamically.

Results Displayed- The  result will be displayed in the results table.
                                The results will be highlighted in GREEN color in the table on the right. Respective cells containing that result data will be highlighted.

OVERFLOW-    If the resultant of the two numbers cannot be represented in given number of "bits", then:
                               1 .There will be an alert generated when an overflow occurs.
                               2 .The last row of the table will be popped up to indicate which column has an overflow .
                               3.The actual values requiring higher number of bits will be displayed in the result table for the user.

Refresh Page-       If you want to add new set of numbers just select them from the table.
                                The Code is written in such a way that there is NO NEED TO RESET WHEN WE NEED TO ADD A NEW SET OF NUMBERS.                                 The Reset Button is only for refreshing the page.
